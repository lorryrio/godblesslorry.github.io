yeoman 简介

yeoman 是一个通用的脚手架系统，开发者可以通过 yeoman 快速创建任意编程语言无关的项目，或者自定义自己项目所需的脚手架工具。
什么是脚手架？脚手架的本质实际就是 node.js 模块。
yeoman 本身并不提供任何脚手架，通过继承一个名为 generators 的基础插件，开发者可以快速创建自己的脚手架项目：

yeoman 对一个脚手架项目有如下几点约定：

  • 项目命名必须使用这样一种形式：generator-xxx
  • package.json 文件中 keywords 关键字必须包含 yeoman-generator 字段:

如此一来，在开发完我们的脚手架并发布安装后，就可以轻松的使用 yo xxx 的方式快速创建项目。
当然，我们并不需要手动执行这些早已约定好的工作，yeoman 提供的 generator-generator 脚手架已经帮我们完成了大部分重复劳动。

templates 目录存放了脚手架创建项目所需的模版，通过在 index.js 中编写自定义的业务逻辑，我们可以快速创建自定义的脚手架工具。

测试

使用 npn link 命令将脚手架工具安装到本地，随后使用 yo xxx （xxx 为脚手架的名字）创建项目，测试脚手架是否能正常工作。

发布

使用 npm publish 发布脚手架工具到 npm 仓库，其他开发者全局安装该脚手架后，也能愉快地创建项目了哦耶～

web性能指标

Posted on 2018-07-04

标准说明

依照W3C Web Performance Working Group 引入的 PerformanceNavigationTiming API的各个属性的介绍，该API将网页的整个加载过程分成不同的阶段，通过它可以帮助我们进行更精准的性能测试打点。在实际计算的过程中采用的 Perfomance API 。具体打点过程如下图所示：

指标介绍

首字节

具体含义：是指在 TCP 协议的三次握手之后，浏览器开始发出 HTTP 请求，然后服务器响应时回复的第一个字节，也就是第一张图中的 responseStart 。
测速点：responseStartTime

可交互

具体含义：等到服务器通过 HTTP 协议将响应全部返回之后，便开始DOM Tree 的构建，完成之后，网页变成可交互状态，到此为止便是网页的可交互时间。用户可以进行正常的事件输入交互操作。
测速点：domInteractive

完全加载

具体含义：DOM Tree 构建完成后，开始加载网页资源，资源完全加载完成后，从加载开始到此为网页的完全加载时间。
测速点：loadEventEnd

解析

具体含义：浏览器开始解析html文档到可交互的时间称为解析时间
计算方式： 解析 = 可交互 - 首字节

加载

具体含义：上述的加载网页资源，这个加载网页资源的过程，为网页的加载时间。
计算方式：加载 = 完全加载 - 可交互

白屏

具体含义：白屏时间是指浏览器从响应用户输入网址地址，到浏览器开始渲染内容的时间
测速点：first-contentful-paint

首屏

具体含义：首屏时间是指浏览器从响应用户输入网络地址，到首屏内容渲染完成的时间。 但是基于加载指标类的数据采集，不足以验证页面的真实性能，结合感官可使用的首屏的时间，来作为首屏时间
测速点：first-screen-time
